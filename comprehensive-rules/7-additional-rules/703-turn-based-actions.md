# 703. Turn-Based Actions

Chapter: 7. Additional Rules
Source: Magic: The Gathering Comprehensive Rules (effective June 19, 2026)

## Rules

<a id="rule-703-1"></a>
703.1. Turn-based actions are game actions that happen automatically when certain steps or phases begin, or when each step and phase ends. Turn-based actions don’t use the stack.

<a id="rule-703-1a"></a>
703.1a Abilities that watch for a specified step or phase to begin are triggered abilities, not turn-based actions. (See rule [603](../6-spells-abilities-and-effects/603-handling-triggered-abilities.md#rule-603), “Handling Triggered Abilities.”)

<a id="rule-703-2"></a>
703.2. Turn-based actions are not controlled by any player.

<a id="rule-703-3"></a>
703.3. Whenever a step or phase begins, if it’s a step or phase that has any turn-based action associated with it, those turn-based actions are automatically dealt with first. This happens before state-based actions are checked, before triggered abilities are put on the stack, and before players receive priority.

<a id="rule-703-4"></a>
703.4. The turn-based actions are as follows:

<a id="rule-703-4a"></a>
703.4a Immediately after the untap step begins, all phased-in permanents with phasing that the active player controls phase out, and all phased-out permanents that the active player controlled when they phased out phase in. This all happens simultaneously. See rule [502.1](../5-turn-structure/502-untap-step.md#rule-502-1).

<a id="rule-703-4b"></a>
703.4b Immediately after the phasing action has been completed during the untap step, if the game has either the day or night designation, it checks to see whether that designation should change. If it’s neither day nor night, this check doesn’t happen. See rule [502.2](../5-turn-structure/502-untap-step.md#rule-502-2).

<a id="rule-703-4c"></a>
703.4c Immediately after the game checks to see if its day or night designation should change during the untap step or, if the game doesn’t have a day or night designation, immediately after the phasing action has been completed during the untap step, the active player determines which permanents they control will untap. Then they untap them all simultaneously. See rule [502.3](../5-turn-structure/502-untap-step.md#rule-502-3).

<a id="rule-703-4d"></a>
703.4d Immediately after the draw step begins, the active player draws a card. See rule [504.1](../5-turn-structure/504-draw-step.md#rule-504-1).

<a id="rule-703-4e"></a>
703.4e In an Archenemy game (see rule [904](../9-casual-variants/904-archenemy.md#rule-904)), immediately after the archenemy’s precombat main phase begins, that player sets the top card of their scheme deck in motion. See rule [701.32](701-keyword-actions.md#rule-701-32).

<a id="rule-703-4f"></a>
703.4f Immediately after a player’s precombat main phase begins, that player puts a lore counter on each Saga enchantment they control with one or more chapter abilities. In an Archenemy game, this happens after the archenemy’s scheme action. See rule [714](714-saga-cards.md#rule-714), “Saga Cards.”

<a id="rule-703-4g"></a>
703.4g Immediately after the action of placing lore counters has been completed, if the active player controls any Attractions, that player rolls to visit their Attractions. See rule [701.52](701-keyword-actions.md#rule-701-52), “Roll to Visit Your Attractions.”

<a id="rule-703-4h"></a>
703.4h Immediately after the beginning of combat step begins, if the game being played is a multiplayer game in which the active player’s opponents don’t all automatically become defending players, the active player chooses one of their opponents. That player becomes the defending player. See rule [507.1](../5-turn-structure/507-beginning-of-combat-step.md#rule-507-1).

<a id="rule-703-4i"></a>
703.4i Immediately after the declare attackers step begins, the active player declares attackers. See rule [508.1](../5-turn-structure/508-declare-attackers-step.md#rule-508-1).

<a id="rule-703-4j"></a>
703.4j Immediately after the declare blockers step begins, the defending player declares blockers. See rule [509.1](../5-turn-structure/509-declare-blockers-step.md#rule-509-1).

<a id="rule-703-4k"></a>
703.4k Immediately after the combat damage step begins, each player in APNAP order announces how each attacking or blocking creature they control assigns its combat damage. See rule [510.1](../5-turn-structure/510-combat-damage-step.md#rule-510-1).

<a id="rule-703-4m"></a>
703.4m Immediately after combat damage has been assigned during the combat damage step, all combat damage is dealt simultaneously. See rule [510.2](../5-turn-structure/510-combat-damage-step.md#rule-510-2).

<a id="rule-703-4n"></a>
703.4n Immediately after the cleanup step begins, if the active player’s hand contains more cards than their maximum hand size (normally seven), they discard enough cards to reduce their hand size to that number. See rule [514.1](../5-turn-structure/514-cleanup-step.md#rule-514-1).

<a id="rule-703-4p"></a>
703.4p Immediately after the active player has discarded cards (if necessary) during the cleanup step, all damage is removed from permanents and all “until end of turn” and “this turn” effects end. These actions happen simultaneously. See rule [514.2](../5-turn-structure/514-cleanup-step.md#rule-514-2).

<a id="rule-703-4q"></a>
703.4q As each step or phase ends, any unspent mana left in a player’s mana pool empties. See rule [500.5](../5-turn-structure/500-general.md#rule-500-5).
